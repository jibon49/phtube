Scope of Var,let and const

In JavaScript, there are three variable declarations: var, let, and const.

Var

  • When var is declared within a function it can be accessible within only function and when declared in global scope it can be accessed anywhere
  • When declared with var its value can be reassign
  • Variable declared with Var can be hoisted

let

  • Variable declared with let are not accessible outside of the block
  • When declared with let value can be reassign and the value can be change
  • let does not hoist like var and its more compatible than var

const

  • Variable declared with const are not accessible outside of the block
  • When declared with const value cannot be reassigned or changed
  • Const variable is always fixed and it does not hoist

Use Cases of null and undefined

null represents the intentional absence of any object value, while undefined represents a variable that has been declared but has not been assigned a value.

null

In JavaScript, null is a special value that represents an empty or unknown value and null is treated as an object. We can check this using the typeof operator.

undefined

If a variable is declared but the value is not assigned, then the value of that variable will be undefined.Functions that do not explicitly return a value return undefined

REST API

A REST API (Representational State Transfer Application Programming Interface) is a set of architectural principles and constraints for designing and interacting with web services.

A REST API is a way for software applications to communicate over the internet. It follows a set of rules that make interactions between different systems easy to understand and use. REST APIs are used to request and exchange data between a client (like a web browser) and a server (where data is stored). Using REST API we can get specified data from a database though using mobile application
